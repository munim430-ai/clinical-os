## 2024-05-24 - React Native Pressable Accessibility for Icon-only Buttons
**Learning:** In React Native, custom interactive elements (like `Pressable` used for floating action buttons or icon-only buttons) do not automatically inherit button accessibility semantics, unlike standard HTML `<button>` tags. This causes screen readers to announce them poorly (e.g., just reading inner text or reading nothing for icons). Additionally, custom toggle states aren't communicated by default.
**Action:** Always add `accessibilityRole="button"` and a descriptive `accessibilityLabel` to icon-only `Pressable` components. If the button has an active/expanded/selected state, communicate it using `accessibilityState={{ expanded: isActive }}` or `accessibilityState={{ selected: isSelected }}`.
