## 2024-05-31 - Missing Accessibility Attributes on AI Floating Components
**Learning:** Found an accessibility issue pattern specific to this app's components, where custom floating AI interactive components (like `AIOrbButton` and `AIAssistantButton`) using `Pressable` or `TouchableOpacity` lacked `accessibilityRole`, `accessibilityLabel`, and `accessibilityState`. Because they use icons or visual cues (e.g. `Sparkles` icon, expanding animations) rather than standard button elements, screen reader users would not have the necessary context about what these buttons do or their current state (expanded vs collapsed).
**Action:** When adding custom interactive UI components that don't use standard button semantics (especially floating action buttons or AI orbs in this project), ALWAYS include `accessibilityRole="button"`, provide a descriptive `accessibilityLabel`, and correctly surface the interactive state via `accessibilityState={{ expanded: isExpanded }}`.
