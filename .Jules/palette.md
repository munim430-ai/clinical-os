## 2024-06-01 - Icon-Only Button Accessibility Pattern\n**Learning:** Icon-only buttons (like `ThemeToggle`) using `Pressable` completely silence screen readers if missing semantic properties, leaving visually impaired users unable to infer their purpose or state. `accessibilityState` should also be added when a button acts as a toggle.\n**Action:** Always pair `accessibilityRole="button"`, `accessibilityLabel` with explicit descriptions (e.g. "Switch to light theme"), and `accessibilityState` (if toggleable) on any interactive element lacking visible text content.
